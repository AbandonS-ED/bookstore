# 订单支付模块设计书

## 1. 背景

当前订单流程中，支付环节是直接标记 `paid` 状态，没有实际的支付流程。需要完善支付模块，实现完整的订单支付流程。

## 2. 现状分析

| 环节 | 当前状态 | 问题 |
|------|----------|------|
| 创建订单 | ✅ 已实现 | 返回 pending 状态 |
| 支付 | ❌ 直接标记 paid | 没有支付申请/回调流程 |
| 发货 | ⚠️ 已实现 | 没有物流单号字段 |
| 收货/完成 | ✅ 已实现 | - |
| 取消 | ⚠️ 已实现 | 没有超时自动取消 |
| 退款 | ❌ 未实现 | - |

## 3. 设计目标

1. 实现完整的支付申请 → 支付回调流程
2. 添加支付记录表，追溯支付流水
3. 实现订单超时自动取消
4. 支持退款流程
5. 记录物流单号

## 4. 数据库设计

### 4.1 新增 payment 表

```sql
CREATE TABLE `payment` (
    `id` BIGINT PRIMARY KEY AUTO_INCREMENT,
    `order_id` BIGINT NOT NULL COMMENT '订单ID',
    `payment_no` VARCHAR(32) NOT NULL UNIQUE COMMENT '支付流水号',
    `payment_method` VARCHAR(20) COMMENT '支付方式(alipay/wechat/card)',
    `amount` DECIMAL(10,2) NOT NULL COMMENT '支付金额',
    `status` VARCHAR(20) DEFAULT 'pending' COMMENT '状态(pending/success/failed/refunded)',
    `paid_time` DATETIME COMMENT '支付时间',
    `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP,
    `update_time` DATETIME ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (`order_id`) REFERENCES `order`(`id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='支付记录表';
```

### 4.2 修改 order 表

新增字段：
```sql
ALTER TABLE `order` ADD COLUMN `payment_id` BIGINT COMMENT '支付记录ID';
ALTER TABLE `order` ADD COLUMN `pay_time` DATETIME COMMENT '支付时间';
ALTER TABLE `order` ADD COLUMN `express_no` VARCHAR(50) COMMENT '物流单号';
ALTER TABLE `order` ADD COLUMN `expire_time` DATETIME COMMENT '订单过期时间';
```

## 5. 订单状态流转

```
                      ┌─────────────┐
                      │   created   │ 创建订单(30分钟过期)
                      └──────┬──────┘
                             │
                  ┌──────────┼──────────┐
                  ↓          ↓          ↓
            ┌─────────┐ ┌──────────┐ ┌──────────┐
            │ expired │ │  paying  │ │ cancelled│
            │ (超时)  │ │  (待支付) │ │ (用户取消)│
            └─────────┘ └────┬─────┘ └──────────┘
                             │
                       支付成功回调
                             ↓
                       ┌──────────┐
                       │   paid   │──────┐
                       └────┬─────┘      │
                            ↓            ↓
                      ┌─────────┐  ┌──────────┐
                      │ shipped │  │ refunded │
                      │ (已发货) │  │ (已退款)  │
                      └────┬────┘  └──────────┘
                           ↓
                      ┌──────────┐
                      │ delivered│
                      │ (已收货)  │
                      └────┬─────┘
                           ↓
                      ┌──────────┐
                      │completed │
                      │ (已完成)  │
                      └──────────┘
```

### 状态说明

| 状态 | 说明 |
|------|------|
| created | 订单已创建，待支付（30分钟超时） |
| paying | 支付申请已提交，等待支付 |
| paid | 支付成功 |
| shipped | 已发货 |
| delivered | 已收货 |
| completed | 已完成 |
| cancelled | 已取消 |
| expired | 已过期（超时未支付） |
| refunded | 已退款 |

## 7. 核心流程

### 7.1 创建订单

```
1. 用户选择书籍，提交订单
2. 校验库存、地址
3. 创建订单，状态为 created
4. 设置 expire_time = now + 30分钟
5. 返回订单信息（包含支付倒计时）
```

### 7.2 申请支付

```
1. 用户点击支付
2. 创建 payment 记录，状态为 pending
3. 更新 order.status = 'paying'
4. 返回支付信息（payment_id、支付金额）
```

### 7.3 模拟支付回调

```
1. 前端调用模拟回调接口
2. 更新 payment.status = 'success'
3. 更新 payment.paid_time
4. 更新 order.status = 'paid'
5. 更新 order.pay_time
6. 关联 order.payment_id
```

### 7.4 订单超时处理

```
1. 定时任务每分钟执行
2. 查询 expire_time < now AND status = 'created' 的订单
3. 更新 status = 'expired'
4. 恢复库存
```

### 7.5 取消订单

```
1. 校验订单状态为 created/paying
2. 恢复库存
3. 更新 status = 'cancelled'
4. 如果有 payment，更新 payment.status = 'failed'
```

### 7.6 退款

```
1. 校验订单状态为 paid/shipped
2. 更新 payment.status = 'refunded'
3. 更新 order.status = 'refunded'
4. 恢复库存
```

## 10. 测试用例

| 用例 | 预期结果 |
|------|----------|
| 创建订单后30分钟未支付 | 订单自动变为 expired 状态，库存恢复 |
| 用户点击支付 | 生成 payment 记录，订单变为 paying |
| 支付成功回调 | payment 和 order 都变为 paid 状态 |
| paying 状态取消订单 | 订单变为 cancelled，payment 变为 failed |
| 已支付订单申请退款 | payment 变为 refunded，order 变为 refunded，库存恢复 |
| 发货时填写物流单号 | express_no 被保存 |

## 11. 注意事项

1. 支付回调需要验证签名（模拟环境下跳过）
2. 订单状态变更需要事务保证一致性
3. 库存操作需要考虑并发问题
4. 定时任务需要保证幂等性
