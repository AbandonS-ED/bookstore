# 拾光书屋 — 网上书店系统

## 一、开场

"我们做的是一个网上书店，名字叫拾光书屋。就是在网上卖书的，你可以想象成一个线上的新华书店。"

技术栈简单说就是：
- 后端用 Java 写的（Spring Boot 框架）
- 前端用 Vue 写的（就是网页界面）
- 数据库用 MySQL（存数据的地方）

---

## 二、功能介绍

### 用户端（买家看到的界面）

1. **首页** — 展示推荐书籍、排行榜、新书上架
2. **搜索和浏览** — 可以按书名、作者搜索，也可以按分类浏览
3. **书籍详情** — 看到书的封面、价格、简介、作者介绍、目录、评论
4. **排行榜** — 四种排行：卖得最好的、评分最高的、最新的、收藏最多的
5. **限时折扣** — 有些书打折，有倒计时，下单自动用折扣价
6. **购物车** — 想买的书先放购物车，统一结算
7. **下单支付** — 选地址、确认订单、模拟支付
8. **订单管理** — 看到自己的订单状态（待发货、已发货、已收货等）
9. **收藏** — 收藏喜欢的书，降价了会提醒
10. **评论** — 给买过的书打分、写评论
11. **AI 书友** — 跟 AI 聊天，让它帮你找书、推荐书、直接加购物车
12. **社区** — 发帖分享读书心得，可以关联书籍，别人可以点赞

### 管理后台（卖家看到的界面）

1. **仪表盘** — 看销售额、订单统计、趋势图
2. **书籍管理** — 添加、修改、删除书籍，上下架
3. **分类管理** — 管理书籍分类
4. **库存管理** — 查看库存数量，调整库存
5. **订单管理** — 查看所有订单，发货、处理退款
6. **退款售后** — 处理用户的退款和售后申请
7. **用户管理** — 查看用户列表，禁用/启用用户
8. **评论管理** — 审核评论，隐藏不当内容
9. **社区管理** — 管理社区帖子，删除违规内容

## 三、技术难点（用大白话解释）

### 难点 1：数字太大，电脑算不了

> 问题：数据库给每条记录生成一个 19 位的编号（比如 1234567890123456789），但 JavaScript 语言最多只能精确处理 15 位数字，超过就会出错。

> 解决：在传给前端之前，把数字转成字符串（就像把 123 变成 "123"），这样就不会丢精度了。我们在三层代码里都做了这个处理。

### 难点 2：两个人同时买最后一本书怎么办

> 问题：库存只剩 1 本书，两个人同时点"下单"，如果不处理，两个人都买到，库存变成 -1。

> 解决：用数据库的原子操作——一条 SQL 同时检查并扣减库存：`库存 = 库存 - 1 WHERE 库存 >= 1`。如果库存不够，这条 SQL 不会执行，订单创建失败。

### 难点 3：订单有很多状态，不能乱

> 问题：一个订单从创建到完成，有很多中间状态（待支付、已支付、已发货、已收货、已取消、已退款等），状态之间的转换必须按顺序来。

> 解决：我们设计了一个状态流转图，代码里严格检查每一步转换。比如只有"已支付"的订单才能发货，只有"已发货"的订单才能确认收货。

### 难点 4：AI 怎么知道书库里有什么书

> 问题：AI 助手要能帮用户搜书、加购物车，但它不知道我们书店里有什么书。

> 解决：我们用了"函数调用"（Function Calling）技术。AI 可以调用我们写的 5 个函数：
> - searchBooks（搜书）
> - getBookDetail（看书的详情）
> - addToCart（加购物车）
> - getCartItems（看购物车里有什么）
> - listAddresses（看收货地址）
>
> AI 根据用户的问题，自动决定调用哪个函数，拿到结果后再回复用户。

### 难点 5：AI 说话太慢怎么办

> 问题：AI 生成回复需要几秒钟，如果等全部生成完再显示，用户会等很久。

> 解决：用 SSE（Server-Sent Events）技术，AI 生成几个字就发几个字，前端实时显示，就像打字一样。

## 四、数据库设计

一共 16 张表，简单理解就是 16 个 Excel 表格：

| 表名 | 作用 |
|------|------|
| user | 存用户信息（账号、密码、角色） |
| book | 存书籍信息（书名、价格、库存） |
| category | 存分类信息（文学、科技、历史等） |
| author | 存作者信息 |
| book_chapter | 存书籍目录 |
| cart | 存购物车 |
| order | 存订单 |
| order_item | 存订单里买了哪些书 |
| payment | 存支付记录 |
| address | 存收货地址 |
| favorite | 存收藏记录 |
| review | 存评论 |
| community_post | 存社区帖子 |
| community_like | 存点赞记录 |
| chat_history | 存 AI 聊天记录 |
| book_excerpt | 存书籍精彩文段 |

## 五、总结

"我们这个项目做了 2 周，3 个人分工完成。主要完成了：
1. 一个完整的网上书店，买家可以浏览、搜索、下单、支付
2. 一个管理后台，卖家可以管理书籍、订单、用户
3. 一个 AI 助手，可以跟用户聊天、推荐书籍
4. 一个社区，用户可以分享读书心得

技术上解决了数字精度、并发超卖、订单状态管理、AI 函数调用等难点。"
